因为网速问题，小网站一直不是很稳定，换各种线路还要加载半天缓冲不过来。我一直在小网站用F12找到视频的m3u8链接下载到本地来看，用ffmpeg, `ffmpeg -i "https://example.m3u8" -c copy example.mp4`，还有的网站直接是视频文件，就可以直接点击下载。

但是用了一段时间之后感觉过于低效，而且这样的命令下载不能断点重传，而且一次下一个ts文件，因为网络不好就下得特别慢，就只是好几个视频一块下慢慢等。

本来浏览器有m3u8下载器的[油猴脚本](https://github.com/Momo707577045/m3u8-downloader)，但是不知道为什么有的时候不会自动检测到，就又找了一个手动下载的[m3u8下载器](https://github.com/nilaoda/N_m3u8DL-RE)。这个下载起来就很快很方便了但是还是要一个个选集。

我想起来之前看到小网站的视频源头，上网搜了一下发现都是从采集站资源站链接的资源，找到一个[聚合网站](https://www.zzzypro.com/)，试了几个发现这种下下来的视频都已经是有水印的了，只能慢慢找看看有什么好点的。

现在用着[索尼资源](https://suonizy.com)和[ok资源](https://okzyw.cc)还不错。

> 下电视剧可以写个脚本使用`N_m3u8DL-RE`按顺序下一整季电视剧。

```powershell
# download.ps1
$urls = Get-Content .\urls.txt
$count = 1

foreach ($url in $urls) {
    if ($url.Trim() -ne "") {
        Write-Host "Downloading file: $count Url: $url" -ForegroundColor Green
        
        # 构建命令
        $saveName = "$count"
        $command = "N_m3u8DL-RE `"$url`" --save-name `"$saveName`""
        
        # 执行命令
        Invoke-Expression $command
        
        $count++
    }
}

Write-Host "Done!" -ForegroundColor Cyan
```
